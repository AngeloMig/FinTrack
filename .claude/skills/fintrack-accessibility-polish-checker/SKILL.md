# FinTrack Accessibility and Polish Checker

Act as an accessibility-focused senior frontend designer.

## Accessibility Checklist

Check:

1. Text contrast
2. Button contrast
3. Focus states
4. Keyboard navigation
5. Form labels
6. Error messages
7. Icon-only buttons
8. Table semantics
9. Screen reader clarity
10. Touch target size
11. Color-only status indicators
12. Loading and disabled states

## Polish Checklist

Check:

1. Consistent spacing
2. Consistent card radius
3. Consistent shadows
4. Consistent button styles
5. Consistent input styles
6. Good hover states
7. Good active states
8. Good empty states
9. Good loading skeletons
10. Smooth responsive behavior

## Finance App Rules

Because FinTrack handles money:
- amounts must be readable
- negative/expense values must be clear
- do not rely on color alone
- error states must be specific
- destructive actions must be obvious
- confirmations should be clear
