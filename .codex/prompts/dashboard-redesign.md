# Dashboard Redesign Prompt

Improve the FinTrack dashboard page.

Important:
- Keep FinTrack as a mobile-first app UI.
- Do not create a wide desktop dashboard.
- Do not widen the main app container.
- Do not add desktop multi-column grid layouts.
- Desktop browser view should remain a centered phone-style app.

Goals:
- make the dashboard feel modern and professional
- make key financial numbers easier to scan
- improve card layout
- improve charts
- improve recent transactions
- improve mobile responsive behavior
- preserve existing data and functionality

Before editing, identify:
- dashboard route/page file
- card components
- chart components
- transaction components

Then improve styling and layout.

Use focused mobile polish:
- spacing inside cards
- typography hierarchy
- touch target size
- transaction row scanning
- chart/card fit inside mobile width
- visible focus states
- no horizontal overflow
