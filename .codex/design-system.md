# FinTrack Design System Reference

## Brand Feel

FinTrack should feel:
- clean
- calm
- secure
- modern
- financially trustworthy
- easy for beginners

## Visual Direction

Use a modern SaaS fintech direction:
- light background
- white cards
- subtle borders
- rounded corners
- clean typography
- useful color, not decorative color
- clear money hierarchy

## Suggested Color Roles

Use existing project colors if already defined. If not, prefer:

- Background: slate-50 or white
- Surface: white
- Border: slate-200
- Main text: slate-900
- Muted text: slate-500 / slate-600
- Primary action: blue, indigo, emerald, or brand color
- Income/success: emerald
- Expense/warning: rose, red, or orange
- Neutral status: slate

## Component Style

### Page Shell

Use generous spacing and centered max width.

### Cards

Cards should use:
- white background
- subtle border
- soft radius
- small shadow
- consistent padding

### Money Values

Money values should be:
- large enough to scan
- aligned consistently
- labeled clearly
- not communicated by color only

### Badges

Use badges for:
- transaction categories
- income/expense
- budget status
- trend direction

### Empty States

Every empty state should explain:
- what is missing
- why it matters
- what action to take next

Example:
"No transactions yet. Add your first transaction to start tracking your spending."

### Loading States

Use skeletons or calm loading indicators for:
- dashboard cards
- charts
- transaction lists
- reports

### Error States

Financial app errors should be specific.

Bad:
"Something went wrong."

Better:
"We couldn't load your transactions. Check your connection and try again."
